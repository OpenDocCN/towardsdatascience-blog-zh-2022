应用程序编程接口(API)是一个概念，它支持并简化软件程序之间的通信。它定义了从应用程序中提取或向应用程序发送[数据](https://databasecamp.de/en/data)的方式和格式。

# 应用编程接口是用来做什么的？

API 通过允许发送、请求、删除或修改信息来实现不同程序和应用程序之间的通信。

这种结构使模块化复杂的软件应用成为可能，它使单个程序部分独立，然后通过编程接口相互通信。这意味着代码的各个功能彼此之间是明确分离的，并且可以很容易地交换或更改。

[比如 Docker](https://databasecamp.de/en/data/docker-basics) 容器就是基于这个原理。这些容器可以在集群中的不同计算机上启动，并执行单独的代码段。为了将它变成一个正常运行的应用程序，它们从其他容器接收数据，并通过 API 将自己的结果发送给下游容器。

# 编程接口有哪些类型？

基本上，有不同的方法将编程接口分组。最常见的情况是，API 根据其可访问性进行区分，即谁被授权访问数据。对以下三种类型进行了区分:

1.  内部 API :这些是专门为公司内部使用的编程接口。它们对外部用户不可见，当然也不可检索。例如，销售部门可以通过 API 访问来自内部客户关系管理(CRM)工具的数据。在任何情况下都不能让外人接触到这一敏感信息。
2.  **合作伙伴/客户 API** :这些编程接口保存了应该被选定的一组人访问的数据。身份验证在这里起着重要的作用，因为它确保用户确实被授权查看信息。例如，一些在线商店提供通过 API 检索客户自己的订单或交付信息的选项，以便客户可以在自己的环境中使用它。在这些应用中，必须确保每个客户只看到与他们的客户号相关的订单。
3.  **公共/开源 API** :这种类型的应用编程接口通过互联网向公众开放。这些数据是公众感兴趣的，这就是运营商提供公众访问的原因。但是，根据使用情况，可能会对数据传输收费，以维护所提供的基础架构。公共 API 包括，例如，谷歌地图接口。

除了通过可访问性进行区分之外，一些源还使用界面定向来创建不同的组:

1.  **面向功能的**:这个组中的接口使程序能够直接访问设备的硬件，以启用那里的特殊功能。例如，可以通过操作系统 API 请求硬件的组件。
2.  **面向文件的**:这允许访问计算机硬盘的请求，并允许输出特定的文件或目录。访问权限通过操作系统进行管理。
3.  **面向对象**:在许多编程语言中，就像在 [Python](https://databasecamp.de/en/python-coding) 中，复杂的应用程序被分成不同的类。面向对象的接口允许这些类(也称为对象)之间的通信，并且比面向函数的接口灵活得多。
4.  **面向协议**:面向协议的应用编程接口完全独立于操作系统和硬件，因为它们使用协议和库工作。

# API 有什么优势？

使用编程接口可以实现无数的优势。其中很大一部分可以归结为这样一个事实:在接口的帮助下，应用程序不再容易出错，更容易维护，因此需要更便宜的基础设施。我们将在接下来的章节中解释这些优势是如何产生的。

应用程序编程接口使得软件应用程序(例如 Docker 应用程序)能够模块化。为此，整个应用程序被重写为更小的子任务，以便这一部分可以自主运行。与上游和下游流程步骤的通信仅通过应用程序编程接口进行。

这种架构使得维护子应用程序、更新它们或者甚至用全新的系统替换它们变得更加容易。

# REST API 是如何工作的？

我们在互联网上看到的内容几乎完全基于 API 查询。在互联网的早期，简单对象访问协议(或简称 SOAP)被用于此目的，它发送 XML 文档。然而，在 21 世纪初，REST 接口(表述性状态转移)成为了 web 服务的基础。

使用 SOAP 的问题在于，服务器通信必须遵守严格的规范，XML 的处理也相当麻烦。另一方面，REST 使用更清晰的 JSON 文件。此外，它提供了灵活的格式，并且在数据传输方面非常高效，这在当今的 web 中扮演着重要的角色。
